# Horizon Learning Publishing Workflow

Version: 1.0

---

# Purpose

This document defines the standard publishing process for all Horizon Learning content.

The goal is consistency, quality, and continuous improvement.

Every lesson should follow this workflow before being considered complete.

---

# Philosophy

Publishing is not the final step.

Publishing is simply the beginning of the lesson's lifecycle.

Every lesson should continue improving over time.

---

# Lesson Lifecycle

## Stage 1 — Idea

Questions to answer:

- What problem does this lesson solve?
- Who is the audience?
- Where does it fit within the learning path?
- What knowledge should the learner already have?

Deliverable:

Lesson outline.

---

## Stage 2 — Research

Gather information from:

- Official documentation
- Technical references
- Standards organizations
- Vendor documentation
- Practical testing

Avoid relying solely on blogs or AI-generated content.

Deliverable:

Verified technical notes.

---

## Stage 3 — Draft

Write the lesson using the Horizon Learning Lesson Template.

Focus on clarity before completeness.

Do not worry about perfect wording on the first draft.

Deliverable:

Complete draft.

---

## Stage 4 — Technical Review

Verify:

- Commands
- Examples
- Screenshots
- Links
- Technical accuracy

Correct anything that is outdated or unclear.

Deliverable:

Technically accurate lesson.

---

## Stage 5 — Beginner Review

Read the lesson as if you have never seen the topic before.

Ask:

- Is any jargon unexplained?
- Does every concept build naturally?
- Would a beginner feel overwhelmed?
- Is the lesson encouraging?

Deliverable:

Beginner-friendly lesson.

---

## Stage 6 — Practical Review

Complete every exercise yourself.

Confirm:

- Instructions work.
- Commands produce expected results.
- Activities are achievable.
- Estimated time is realistic.

Deliverable:

Verified practice section.

---

## Stage 7 — Mobile Review

Read the lesson on a phone.

Check:

- Paragraph length
- Tables
- Images
- Callouts
- Code blocks
- Navigation

Deliverable:

Mobile-friendly lesson.

---

## Stage 8 — Publish

Publish to:

- Website
- Repository

Update:

- Navigation
- Changelog
- Downloads (if applicable)

Deliverable:

Published lesson.

---

## Stage 9 — Resource Generation

If applicable, generate:

- Printable PDF
- Interactive PDF
- Worksheets
- Cheat sheets

Deliverable:

Supporting resources.

---

## Stage 10 — Periodic Review

Review lessons periodically.

Check for:

- Broken links
- Updated software
- New screenshots
- Better explanations
- Community feedback

Deliverable:

Updated lesson.

---

# Continuous Improvement

No lesson is ever considered permanently finished.

As technology changes, Horizon Learning changes.

Every revision should leave the lesson clearer, more accurate, and more useful than before.
