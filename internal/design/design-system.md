# Horizon Learning Design System

Version: 1.0

---

# Purpose

This document defines the visual language used throughout Horizon Learning.

The goal is consistency, readability, accessibility, and long-term maintainability.

A learner should recognize a Horizon Learning lesson even without seeing the logo.

---

# Design Philosophy

Our design should feel:

- Calm
- Professional
- Modern
- Friendly
- Uncluttered
- Mobile-first

Avoid visual noise.

Good education should reduce cognitive load, not increase it.

---

# Core Principles

## Clarity Before Decoration

Visual elements should improve understanding.

Never add graphics simply to make a page look interesting.

Every visual should teach something.

---

## Mobile First

Most learners will probably read lessons on a phone.

Design decisions should prioritize:

- Short paragraphs
- Comfortable spacing
- Simple navigation
- Readable code blocks
- Large touch targets

Desktop layouts should enhance the mobile experience, not replace it.

---

## Consistency

Every lesson should feel like it belongs to the same platform.

Headings, spacing, callouts, diagrams, and tables should follow the same style.

---

# Color Philosophy

Colors should communicate meaning.

Avoid decorative colors.

Recommended meanings:

Blue:
Information

Green:
Success

Yellow:
Tips

Red:
Warnings

Gray:
Reference material

Never rely on color alone to communicate important information.

---

# Admonitions

Use Material for MkDocs admonitions consistently.

!!! note

Used for:

- Definitions
- Background information
- Additional context

---

!!! tip

Used for:

- Practical advice
- Study tips
- Best practices

---

!!! warning

Used for:

- Safety
- Potential mistakes
- Destructive actions

---

!!! success

Used for:

- Completed milestones
- Positive reinforcement

---

!!! info

Used for:

- Interesting technical background
- Historical information
- Optional reading

---

# Paragraph Length

Maximum:

3–5 sentences.

Longer explanations should be broken into sections.

---

# Lists

Prefer bullet lists over large paragraphs whenever possible.

Lists are easier to scan on mobile devices.

---

# Tables

Tables should:

- Fit typical mobile screens.
- Have clear headers.
- Contain only essential information.

---

# Code Blocks

Every command should be copyable.

Avoid screenshots of terminal output whenever possible.

Prefer real text.

---

# Images

Images should:

- Teach something.
- Include alt text.
- Be high resolution.
- Avoid unnecessary annotations.
- Match the current software version.

---

# Diagrams

Diagrams should:

- Explain concepts.
- Use consistent spacing.
- Be understandable without color.
- Print well.
- Remain readable on phones.

---

# Screenshots

Capture only what is necessary.

Highlight important areas.

Avoid personal information.

Avoid clutter.

---

# Accessibility

Every page should be usable by learners with different needs.

Requirements:

- Good contrast
- Descriptive headings
- Alt text
- Keyboard-friendly navigation
- Color-independent explanations

---

# Long-Term Goal

The Horizon Learning visual language should become recognizable through clarity, consistency, and thoughtful educational design—not through flashy graphics or branding.
