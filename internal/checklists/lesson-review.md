# Horizon Learning Lesson Review Checklist

Version: 1.0

---

# Purpose

This checklist is used before any lesson is considered complete.

The goal is consistency, technical accuracy, and an excellent learner experience.

---

# Technical Review

## Accuracy

- [ ] Technical concepts are correct.
- [ ] Commands have been tested.
- [ ] Command output matches current software versions.
- [ ] Examples are realistic.
- [ ] Screenshots (if present) match current software.
- [ ] Links have been verified.
- [ ] No deprecated commands are used.

---

# Educational Review

## Learning Flow

- [ ] Lesson follows the Horizon Lesson Template.
- [ ] Learning objectives are measurable.
- [ ] Goal is clearly explained.
- [ ] "Why This Matters" explains real-world value.
- [ ] Plain-English explanation comes before technical details.
- [ ] New terms are defined before use.
- [ ] Information builds logically.
- [ ] No unnecessary jargon.

---

## Practice

- [ ] Practical exercise included.
- [ ] Activity reinforces the lesson.
- [ ] Instructions are clear.
- [ ] Exercise can be completed safely.
- [ ] Estimated completion time is realistic.

---

## Reflection

- [ ] Reflection questions encourage understanding.
- [ ] Lesson promotes critical thinking.
- [ ] Reflection matches lesson objectives.

---

# Safety Review

- [ ] Activities are ethical.
- [ ] Learners are reminded to practice only on systems they own or have permission to test.
- [ ] Potential risks are explained.
- [ ] Dangerous commands are clearly identified.
- [ ] Destructive commands are avoided unless absolutely necessary.

---

# Writing Review

- [ ] Reading level is beginner friendly.
- [ ] Paragraphs are short.
- [ ] Headings are descriptive.
- [ ] Lists improve readability.
- [ ] Tables are easy to understand.
- [ ] Grammar and spelling checked.
- [ ] Tone is encouraging and professional.

---

# Mobile Review

- [ ] Comfortable to read on a phone.
- [ ] Code blocks fit typical screens.
- [ ] Tables remain readable.
- [ ] Images scale correctly.
- [ ] Admonitions display properly.

---

# PDF Review

- [ ] Printable without formatting problems.
- [ ] Page breaks make sense.
- [ ] Images remain readable.
- [ ] Code blocks fit the page.
- [ ] Checklists print correctly.

---

# Accessibility Review

- [ ] Images include alt text.
- [ ] Information is not conveyed by color alone.
- [ ] Headings follow a logical hierarchy.
- [ ] Link text is descriptive.
- [ ] Tables include headers.

---

# Publishing Review

- [ ] YAML metadata completed.
- [ ] Navigation updated.
- [ ] Changelog updated (if appropriate).
- [ ] Downloads updated (if appropriate).
- [ ] Internal links verified.
- [ ] External links verified.

---

# Final Question

Before publishing, ask:

"If this were the first Horizon Learning lesson someone ever read, would I be proud for it to represent the platform?"

If the answer is "no," continue improving the lesson.
