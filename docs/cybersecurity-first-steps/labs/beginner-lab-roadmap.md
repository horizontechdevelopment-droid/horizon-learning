# Beginner Lab Roadmap

## Goal

Understand what to practice next after the first beginner labs.

## Plain-English explanation

Cybersecurity is learned through safe practice.

The best labs are controlled, legal, and designed for learning.

Do not practice on random real systems.

## Stage 1: Computer basics

Practice:

- Opening the terminal
- Moving through folders
- Creating files
- Reading command help
- Understanding file paths

Suggested lessons:

- Terminal Basics
- Files and Folders
- Useful Commands

## Stage 2: Networking basics

Practice:

- Finding your IP address
- Finding your router IP
- Understanding ports
- Understanding DNS
- Understanding Wi-Fi security settings

Suggested lessons:

- Networking Basics
- Wi-Fi Basics
- IP Addresses and Ports

## Stage 3: Safe local scans

Practice:

- Scanning your own computer
- Scanning your own router
- Reading open ports
- Writing down findings

Suggested tools:

- Nmap

## Stage 4: Traffic observation

Practice:

- Starting a Wireshark capture
- Stopping a capture
- Filtering DNS traffic
- Filtering HTTP traffic
- Recognizing normal background traffic

Suggested tools:

- Wireshark

## Stage 5: Purpose-built training labs

Once the basics are comfortable, move to training platforms that are designed for legal practice.

Good lab types include:

- Local virtual machines
- Capture The Flag challenges
- Web app practice labs
- Beginner networking labs
- Spare router lab networks

## Stage 6: Future Wi-Fi lab

More advanced Wi-Fi security practice should only be done with a spare router or isolated lab network.

Do not practice Wi-Fi attack techniques on neighbors, public Wi-Fi, school networks, work networks, or anyone else's network.

## Suggested beginner milestones

- [ ] Understand the safety rules
- [ ] Boot into Kali
- [ ] Open the terminal
- [ ] Run basic Linux commands
- [ ] Find your own IP address
- [ ] Find your router IP
- [ ] Run a local Nmap scan
- [ ] Open Wireshark
- [ ] Complete the Home Wi-Fi Security Check
- [ ] Write a simple findings report

## What a simple findings report looks like

Use this format:

    Date:
    Lab:
    Target:
    Tool used:
    What I expected:
    What I found:
    What I learned:
    Questions for later:

## What to learn after this guide

Good next topics:

- Linux basics
- Networking fundamentals
- Web security basics
- Python basics
- Digital privacy
- Password safety
- Blue team monitoring
- Capture The Flag practice

## Stop and check

You are ready for the next stage when:

- [ ] You can explain what you are testing.
- [ ] You know you have permission.
- [ ] You can write down what you did.
- [ ] You can explain the result in plain English.
