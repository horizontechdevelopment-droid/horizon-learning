# Safety and Permission

## Goal

Understand where cybersecurity learning is allowed and where it can get you into trouble.

## Plain-English explanation

Cybersecurity tools are powerful.

The same tool that helps you check your own network can also be misused against someone else's network.

This guide is only for legal, authorized learning.

Cybersecurity is not about attacking people. It is about understanding systems, finding weaknesses, and learning how to fix or prevent problems.

## The main rule

Only test something if:

- You own it
- You control it
- Or you have clear permission from the owner

If the answer is not clearly yes, stop.

## Safe places to practice

You can practice on:

- Your own computer
- Your own Wi-Fi network
- Your own router
- A spare router used only for practice
- Local virtual machines
- Training labs made for cybersecurity practice

## Do not test

Do not scan, attack, or probe:

- Someone else's Wi-Fi
- School networks
- Work networks
- Public Wi-Fi
- Random websites
- Friends' devices without clear permission
- Neighbor networks
- Any system you do not own or control

## Why this matters

Some cybersecurity tools send traffic that can look suspicious or hostile.

Even if your intent is harmless, using those tools on systems you do not own can cause serious problems.

## Safe beginner mindset

A good beginner question is not:

“Can I hack this?”

A better question is:

“Do I understand how this works, and am I allowed to test it?”

## Stop and check

Before using any tool, ask:

- [ ] Do I own this system or network?
- [ ] Do I have permission?
- [ ] Do I understand what the tool is about to do?
- [ ] Am I prepared to stop if something looks wrong?

If any answer is no, stop.

Continue to the learning path.
